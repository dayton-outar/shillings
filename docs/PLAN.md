## Extraction Plan

Use this extraction method for each chapter:

- Modernise the content into an intuitive narrative that a high school graduate can understand.
- Keep the original heading ideas, but remove section numbers.
- Ignore document preambles, opening learning objectives, cross-references to other chapters or sections, and summaries.
- Preserve important bullet lists, but make each bullet shorter and easier to scan.
- Extract table illustrations into Markdown tables.
- Replace images, charts, and figures with clear placeholders.
- For every image placeholder, state the source PDF file name and the image reference number from the PDF, such as `Fig. X`.
- Put examples, insights, and case applications into indented quotes using `>`.
- Put mathematical symbols and formulas inside LaTeX using the form `$\large{...}$`.
- Keep the result platform-agnostic unless a concept itself requires a technology-neutral explanation of automation, data, or systems.
- Make each chapter useful as project knowledge, not merely as a book note.
